💡 1. Core Idea

We propose a trust-calibrated multi-agent scientific deliberation framework designed to mitigate sycophantic consensus collapse in LLM reasoning. Rather than allowing agents to converge via ungrounded social pressure, our system introduces a dynamic, evidence-weighted aggregation mechanism. Agents whose claims align with progressive retrieval of scientific literature maintain higher trust weights, preventing hallucinating majorities from suppressing correct minority agents.

🎯 Core Research Contributions
  • Trust-Calibrated Evidence-Grounded Aggregation: Dynamic evidence-weighted anti-sycophancy aggregation (our primary systems contribution).
  • Progressive Retrieval for Scientific Deliberation: A supporting architecture enabling iterative verification of atomic claims.
  • Sycophancy-Focused Evaluation Framework: Introducing robust metrics including CCR, MPR, and ECR to elevate evaluation beyond standard accuracy.

🔍 2. Problem Statement

2.1 The Real-World Problem

Scientific reasoning is selected as the target domain because it is evidence-intensive, high-stakes, and highly sensitive to hallucinated consensus — making it the ideal stress-test for sycophantic failure modes in multi-agent systems.

Multi-agent debate has emerged as a leading paradigm for improving LLM reasoning reliability (Du et al., 2023; Liang et al., 2023). The intuition is sound: multiple independent models challenge each other's reasoning, converging on correct answers through dialectical pressure. However, the paradigm suffers from a well-documented and dangerous failure mode — agentic sycophancy.

Sycophancy manifests when agents abandon correct initial positions to conform with a confidently-stated but incorrect majority opinion. Perez et al. (2023) demonstrated sycophancy rates of 40–55% on complex reasoning tasks when a planted incorrect majority opinion was introduced. Wei et al. (2024) further showed this behaviour persists even in the largest frontier models.

2.2 Engineering Complexity

From a systems perspective, this requires solving four interdependent challenges:

Challenge A: Real-time Verification

Decompose each agent's claim into atomic propositions and retrieve supporting/contradicting evidence within the debate turn latency budget (<10s per turn).

Challenge B: Trust Stability

Design a trust update rule that avoids degenerate equilibria — all trust collapsing to zero, or trust becoming dominated by the first correct agent regardless of subsequent performance.

Challenge C: Behavioural Effectiveness

Ensure trust weights actually change model outputs, not just appear in the prompt context and get ignored by the model's attention mechanism.

Challenge D: Heterogeneous Agents

Extend the framework to agents from different model families (Qwen-3.5, Mistral Small 4, Phi-4-Reasoning) — trust calibration must be model-agnostic.

⚠ Explicit Research Gap

Consensus frequency is not equivalent to empirical correctness under adversarial sycophancy conditions. No existing framework provides a formally specified, evidence-grounded trust mechanism that dynamically re-weights agent influence based on the verifiability of their claims against an external corpus. The gap is not "multi-agent debate doesn't work" — it is that existing systems lack the algorithmic infrastructure to penalize hallucinating agents and reward evidence-aligned agents within the same debate session.

🏗️ 4. Proposed Architecture

4.1 Adaptive Debate Triggering

Not every query warrants a full multi-agent debate. Our system uses an adaptive confidence estimator to gate deliberation:

Input → Single-Agent Pass → Confidence Estimator → IF uncertain → Trigger Multi-Agent Debate

This improves computational realism, reduces unnecessary inference cost, and improves publication maturity by demonstrating practical engineering judgment.

4.2 Debate Agent Pool

N agents (N = 3 by default) instantiated from open-weight models (Qwen-3.5, Mistral Small 4, Phi-4-Reasoning — one each). Each agent maintains: (a) its current position on the debate topic, and (b) a trust vector T ∈ ℝN representing its assigned credibility weight. At initialisation, all trust values are uniform: Ti,j = 1/N.

4.3 Source-Partitioned RAG Evidence Retrieval

After each debate turn, every claim made by the active agent is decomposed into atomic propositions (using an LLM prompt with structured JSON output) and queried against partitioned source corpora to reduce correlated retrieval hallucinations:

  • Agent A → PubMed (biomedical literature)
  • Agent B → ArXiv (preprints & CS/Physics)
  • Agent C → Semantic Scholar (cross-domain)

A cross-encoder reranker returns top-K evidence with a relevance score per claim. Claims with no supporting evidence above a threshold θ are flagged as abstentions and excluded from trust updates.

4.4 Trust Calibration Function (Core Contribution)

For agent i at debate round t, the trust update is:

Ti(t+1) = Ti(t) · σ( α · Vi(t) − β · Hi(t) )

Variables

  • Vi — Verification score: proportion of claims supported by evidence
  • Hi — Hallucination penalty: proportion of claims contradicted
  • α, β — Tunable hyperparameters (sweep range: 0.5–2.0)
  • σ — Sigmoid function, bounds trust in (0, 1)

Design Properties

  • Trust never collapses to zero (sigmoid lower bound > 0)
  • Trust floor Tmin = 0.1 prevents agent silencing
  • Trust ceiling Tmax = 0.9 prevents single-agent dominance
  • Session-scoped — resets between questions

4.5 Properties of Trust Dynamics

To ensure stability, the trust calibration mechanism exhibits the following theoretical properties. These are theoretical intuitions rather than formal convergence guarantees, consistent with the scope of a rigorous systems paper.

  • Bounded Trust Intervals: Trust is strictly bounded within [Tmin, Tmax] to prevent irreversible suppression or complete dominance.
  • Monotonicity: If an agent repeatedly produces evidence-supported claims, its trust converges monotonically toward Tmax.
  • Resistance to Collapse: Contradicted claims produce bounded trust decay. The trust floor prevents irreversible suppression of minority agents.
  • Minority Preservation Intuition: Even a single correct agent can outweigh a hallucinating majority if its evidence-grounded trust exceeds the combined decayed trust of the majority.
  • Probabilistic Interpretation: Trust Ti can be interpreted as a bounded empirical estimate of agent i's conditional correctness under evidence-grounded verification — i.e., P(agent i is correct | evidence retrieved so far).

4.5 Trust-Weighted Adjudication

Final answer is determined by trust-weighted aggregation over agent final positions, not majority vote. This is conceptually analogous to weighted ensemble methods but applied at the reasoning trace level — each agent's complete argument chain is weighted by its accumulated trust score Ti(K) at round K.

📌 10.5 Scope Boundaries

🛑 What This Work Does Not Claim

This work focuses specifically on scientific deliberation under adversarial consensus pressure and does not attempt to address general-purpose LLM alignment. Concretely, it does not claim to:

— Eliminate hallucinations (only reduce and detect them).
— Provide formal convergence proofs (only theoretical intuitions and empirical evidence).
— Generalise to all domains without retrieval infrastructure.
— Address long-term agent memory, planning, or general reasoning alignment.

🔍 10.6 Planned Failure Analysis

Systematic failure analysis is planned as part of Phase 4 (Apr–May 2027). Known risk scenarios and planned diagnostic approaches:

Correlated Hallucination

If all N agents share a common pre-training error, the trust function cannot detect it — all agents agree incorrectly. Diagnostic: compare homogeneous vs. heterogeneous agent configurations.

Retrieval Failure / Evidence Sparsity

For very recent or niche questions, no supporting evidence exists in corpora. Diagnostic: track abstention rate; report performance stratified by retrieval success rate.

Trust Collapse

Repeated contradictions early in a session may cause trust to approach Tmin for all agents. Diagnostic: monitor trust variance across rounds; trust floor prevents full collapse.

Adversarial Consensus

Two agents may converge on a confident but unverifiable claim, overwhelming a correct third agent. Diagnostic: track MPR under 2-vs-1 adversarial injection conditions.

1. Problem & Motivation

What exact problem are you solving? Most Important
We address the problem of agentic sycophancy in Multi-Agent Debate (MAD) systems, where LLM agents abandon their correct initial positions to conform with a confidently-stated but incorrect majority opinion.

While debate systems aim to improve reasoning, sycophancy completely undermines this by causing the system to converge on confident hallucinations rather than ground truth. This is a critical failure mode in domains like: scientific reasoning, healthcare, and high-stakes decision support.
Why is this problem important in real-world systems?
In high-stakes environments like clinical diagnosis, drug research, and scientific peer review, an AI system that converges on a wrong answer due to social pressure — not evidence — is more dangerous than one that simply doesn't know.

The specific danger: if a hallucinating majority of agents outvotes a correct minority agent, the final system output is confidently wrong. Standard MAD has no mechanism to detect or prevent this.
Why do current LLMs fail in scientific reasoning?
Current LLMs rely on parametric knowledge (frozen at training time) and probabilistic next-token prediction. They lack:

1) Real-time evidence retrieval — they cannot verify claims against new literature.
2) Claim-level grounding — they produce full paragraph outputs without per-claim verification.
3) Social resistance — when framed with a confident majority, they conform rather than reason independently.

Huang et al. (2024) formally showed that LLMs cannot reliably self-correct reasoning without external feedback — which is the core motivation for our RAG-grounded external verification module.
Why is confidence calibration critical?
ECE (Expected Calibration Error) measures whether a model's confidence scores match its actual empirical accuracy. A model with poor calibration says "90% confident" when it is right only 50% of the time.

In our system, the trust score Ti must itself be calibrated — it is meaningless to weight an agent highly if the trust score does not reflect its true probability of being correct. That is why we evaluate trust quality using ECE < 0.06 as a target metric.

2. Research Gap & Novelty

What is the research gap in existing literature? Most Important
Existing multi-agent debate systems improve reasoning quality, but most rely on: majority voting or equal-weighted aggregation.

They lack a formal mechanism for: trust calibration, evidence reliability scoring, and dynamic agent weighting.
What is your core novelty?
The core novelty is the Trust Calibration Function:

Ti(t+1) = Ti(t) · σ(α · Vi(t) − β · Hi(t))

This is one of the first evidence-grounded, trust-calibrated frameworks specifically targeting sycophantic consensus collapse for multi-agent debate systems. No existing MAD framework (Du et al., Chan et al., Zhang et al.) contains any form of differential agent weighting based on real-time evidence retrieval. This gap is both clear and consequential.
How is your work different from existing debate systems?
Key differences vs. each baseline:

vs. Du et al. (MAD): They use majority vote; we use trust-weighted adjudication — a hallucinating majority cannot override an evidence-grounded minority.

vs. Chan et al. (ChatEval): They assign fixed roles per round; we use dynamic trust weights that evolve based on empirical claim performance.

vs. Zhang et al. (Static Judge): Their judge selects the most convincing argument; ours selects the most evidence-grounded argument via formal trust scores.

vs. Gou et al. (CRITIC): Single-agent tool-use; we operate at the multi-agent session level with cross-agent trust dynamics.
What is the key contribution of this research?
Primary (Algorithmic): Trust-Calibrated Evidence-Grounded Aggregation — one of the first evidence-grounded, trust-calibrated frameworks specifically targeting sycophantic consensus collapse in multi-agent debate.

Secondary (System): A complete RAG-grounded claim verification pipeline using Semantic Scholar + cross-encoder reranker operating within a <10s latency budget per turn.

Tertiary (Evaluation): A reproducible sycophancy measurement protocol using planted incorrect majority injection — applicable to any future multi-agent system.

3. Research Formulation & Engineering Complexity

What is your primary research question? Most Important
Our primary research question is:

"Can trust-calibrated evidence-grounded aggregation reduce sycophantic consensus collapse?"

This is the core of the paper. Everything else (progressive retrieval, source partitioning) is secondary supporting architecture.
Why is this considered a complex engineering problem?
The project requires solving four tightly interdependent engineering challenges simultaneously:

Challenge A — Real-time Verification: Decompose claims into atomic propositions and retrieve supporting evidence within a <10s latency budget per debate turn.

Challenge B — Trust Stability: Prevent degenerate equilibria where trust collapses to zero or becomes permanently dominated by the first correct agent. Solved via Tmin=0.1 and Tmax=0.9 bounds.

Challenge C — Behavioural Effectiveness: Ensure trust weights actually change model outputs and are not silently ignored by the LLM's attention mechanism. This requires careful prompt engineering around trust injection.

Challenge D — Heterogeneous Agents: The trust calibration must work across Qwen-3.5, Mistral Small 4, and Phi-4-Reasoning without model-specific tuning — i.e., it must be model-agnostic.
Why is this research and not just prompt engineering?
Prompt engineering would mean writing a better system prompt and hoping the model behaves differently.

Our contribution is fundamentally different:

1) Formal mathematical specification — the trust update equation Ti(t+1) = Ti(t) · σ(αV − βH) is a computable function with measurable inputs and outputs.

2) Ablation separability — we can independently disable the RAG module, the trust function, or both, and measure each component's isolated contribution.

3) Empirical falsifiability — we make a specific quantitative claim (≥20% sycophancy reduction) that can be confirmed or refuted, distinguishing this as hypothesis-driven research.

4. System Design & Architecture

How does your system work end-to-end? Most Important
The framework operates as a uniform multi-agent debate with a continuous trust-calibration loop:

1. Debate Pool: N agents debate a scientific problem, starting with equal trust weights.

2. RAG Verification: After each turn, claims are decomposed into atomic propositions and queried against Semantic Scholar APIs to retrieve evidence.

3. Trust Update: Each agent's trust score is mathematically updated (penalised for contradicted claims, rewarded for supported claims).

4. Adjudication: The final answer is determined by a trust-weighted aggregation of all agents' final positions, rather than a simple majority vote.
Why use multiple agents instead of one LLM?
A single LLM is bounded by its own training distribution and internal biases. It cannot challenge itself with a genuinely independent perspective.

Multiple agents provide: epistemic diversity — different models have different knowledge gaps and biases, so disagreements are informative signals, not noise.

Crucially, when one agent is tempted to conform (sycophancy), our trust calibration function penalises that agent proportionally — something impossible in a single-agent setting where there is no "other agent" to ground against.
Why is debate better than self-consistency prompting?
Self-consistency generates multiple answers from the same reasoning process.

Debate introduces: disagreement, critique, counter-arguments, and iterative verification.

This improves hallucination detection and reasoning robustness.
How are trust scores calculated?
Trust scores are computed via a formal mathematical update rule after each round:

- Verification Score (V): Proportion of claims explicitly supported by RAG evidence.
- Hallucination Penalty (H): Proportion of claims explicitly contradicted.
- Update Function: The previous trust is multiplied by a sigmoid function of (αV - βH), bounding trust in (0.1, 0.9).

This ensures trust is session-scoped and dynamically prevents hallucinating agents from dominating the debate.

7. Limitations, Ethics & Future Work

What are the limitations of your approach? Most Important
We identify four specific failure modes and mitigations:

1) Correlated agent bias: If all N agents share the same pre-training distribution, they may all hallucinate the same thing. The trust function cannot detect this because all agents agree. Mitigation: heterogeneous model families (Qwen-3.5 + Mistral Small 4 + Phi-4-Reasoning).

2) RAG retrieval noise: If the Semantic Scholar API returns low-quality or irrelevant evidence, the trust signal degrades. Mitigation: cross-encoder reranker + citation-count filter (>10 citations).

3) Domain coverage gaps: For very recent or niche research questions, no supporting evidence may exist in PubMed/ArXiv. Mitigation: claims flagged as “unverifiable” are excluded from trust updates; treated as abstentions.

4) Latency overhead: RAG + reranking adds compute per debate turn. Mitigation: 4-bit quantization, vLLM batched inference, <10s per-turn budget enforced.
Can debate systems still generate incorrect answers?
Yes — our system reduces but cannot eliminate hallucinations.

The key claim is relative improvement: ≥20% sycophancy reduction vs. standard MAD, and <8% hallucination rate vs. the ungrounded MAD baseline. We explicitly acknowledge this domain-of-applicability boundary in the paper — on trivial questions where all agents agree, our system provides no benefit and we report this honestly.
What ethical concerns exist in your system?
Risk 1 — Overconfident wrong outputs: Trust calibration reduces but cannot eliminate the risk that a high-trust agent is wrong. Mitigation: ECE <0.06 target ensures trust scores reflect true accuracy.

Risk 2 — Biased evidence retrieval: Semantic Scholar may over-represent English-language or well-funded research institutions. Mitigation: citation-count and venue-quality filters.

Risk 3 — Deployment misuse: End-users may treat system outputs as definitive scientific truth. Mitigation: outputs clearly labelled with trust ECE and evidence source links.

Importantly, all datasets are fully public with no IRB, HIPAA, or personal data requirements.
How can this work be extended in future?
Extensions grounded in the current architecture:

1) Cross-session trust memory: Rather than resetting trust per question, accumulate trust across sessions to build long-term agent reputation scores.

2) Multimodal evidence grounding: Extend RAG to retrieve figures, tables, and equations from papers — not just text abstracts — for richer claim verification.

3) Adversarial robustness: Stress-test the trust function against coordinated hallucination attacks (two agents colluding on a false claim).

4) Test-time adaptation: Fine-tune agent models on domain-specific corpora to reduce the RAG latency bottleneck in specialised fields like rare disease diagnosis.
How will you conduct human evaluation?
Human evaluators (20–30 samples, graduate students or domain experts) will assess each system output on 5-point Likert scales for:

1) Factual Grounding — Are claims supported by retrievable evidence?
2) Hallucination Severity — How frequently does the output contain unsupported assertions?
3) Reasoning Coherence — Is the deliberation logically consistent?
4) Trustworthiness — Would a domain expert rely on this output?

Inter-annotator agreement will be measured using Cohen’s κ.
