💡 1. Core Idea

We propose a trust-calibrated multi-agent scientific deliberation framework designed to mitigate sycophantic consensus collapse in LLM reasoning. Rather than allowing agents to converge via ungrounded social pressure, our system introduces a dynamic, evidence-weighted aggregation mechanism. Agents whose claims align with progressive retrieval of scientific literature maintain higher trust weights, preventing hallucinating majorities from suppressing correct minority agents.

🎯 Core Research Contributions
  • Trust-Calibrated Evidence-Grounded Aggregation: Dynamic evidence-weighted anti-sycophancy aggregation (our primary systems contribution).
  • Progressive Retrieval for Scientific Deliberation: A supporting architecture enabling iterative verification of atomic claims.
  • Sycophancy-Focused Evaluation Framework: Introducing robust metrics including CCR, MPR, and ECR to elevate evaluation beyond standard accuracy.

🔍 2. Problem Statement

2.1 The Real-World Problem

Scientific reasoning is selected as the target domain because it is evidence-intensive, high-stakes, and highly sensitive to hallucinated consensus — making it the ideal stress-test for sycophantic failure modes in multi-agent systems.

Multi-agent debate has emerged as a leading paradigm for improving LLM reasoning reliability (Du et al., 2023; Liang et al., 2023). The intuition is sound: multiple independent models challenge each other's reasoning, converging on correct answers through dialectical pressure. However, the paradigm suffers from a well-documented and dangerous failure mode — agentic sycophancy.

Sycophancy manifests when agents abandon correct initial positions to conform with a confidently-stated but incorrect majority opinion. Perez et al. (2023) demonstrated sycophancy rates of 40–55% on complex reasoning tasks when a planted incorrect majority opinion was introduced. Wei et al. (2024) further showed this behaviour persists even in the largest frontier models.

2.2 Engineering Complexity

From a systems perspective, this requires solving four interdependent challenges:

Challenge A: Real-time Verification

Decompose each agent's claim into atomic propositions and retrieve supporting/contradicting evidence within the debate turn latency budget (<10s per turn).

Challenge B: Trust Stability

Design a trust update rule that avoids degenerate equilibria — all trust collapsing to zero, or trust becoming dominated by the first correct agent regardless of subsequent performance.

Challenge C: Behavioural Effectiveness

Ensure trust weights actually change model outputs, not just appear in the prompt context and get ignored by the model's attention mechanism.

Challenge D: Heterogeneous Agents

Extend the framework to agents from different model families (Qwen-3.5, Mistral Small 4, Phi-4-Reasoning) — trust calibration must be model-agnostic.

⚠ Explicit Research Gap

Consensus frequency is not equivalent to empirical correctness under adversarial sycophancy conditions. No existing framework provides a formally specified, evidence-grounded trust mechanism that dynamically re-weights agent influence based on the verifiability of their claims against an external corpus. The gap is not "multi-agent debate doesn't work" — it is that existing systems lack the algorithmic infrastructure to penalize hallucinating agents and reward evidence-aligned agents within the same debate session.

🏗️ 4. Proposed Architecture

4.1 Adaptive Debate Triggering

Not every query warrants a full multi-agent debate. Our system uses an adaptive confidence estimator to gate deliberation:

Input → Single-Agent Pass → Confidence Estimator → IF uncertain → Trigger Multi-Agent Debate

This improves computational realism, reduces unnecessary inference cost, and improves publication maturity by demonstrating practical engineering judgment.

4.2 Debate Agent Pool

N agents (N = 3 by default) instantiated from open-weight models (Qwen-3.5, Mistral Small 4, Phi-4-Reasoning — one each). Each agent maintains: (a) its current position on the debate topic, and (b) a trust vector T ∈ ℝN representing its assigned credibility weight. At initialisation, all trust values are uniform: Ti,j = 1/N.

4.3 Source-Partitioned RAG Evidence Retrieval

After each debate turn, every claim made by the active agent is decomposed into atomic propositions (using an LLM prompt with structured JSON output) and queried against partitioned source corpora to reduce correlated retrieval hallucinations:

  • Agent A → PubMed (biomedical literature)
  • Agent B → ArXiv (preprints & CS/Physics)
  • Agent C → Semantic Scholar (cross-domain)

A cross-encoder reranker returns top-K evidence with a relevance score per claim. Claims with no supporting evidence above a threshold θ are flagged as abstentions and excluded from trust updates.

4.4 Trust Calibration Function (Core Contribution)

For agent i at debate round t, the trust update is:

Ti(t+1) = Ti(t) · σ( α · Vi(t) − β · Hi(t) )

Variables

  • Vi — Verification score: proportion of claims supported by evidence
  • Hi — Hallucination penalty: proportion of claims contradicted
  • α, β — Tunable hyperparameters (sweep range: 0.5–2.0)
  • σ — Sigmoid function, bounds trust in (0, 1)

Design Properties

  • Trust never collapses to zero (sigmoid lower bound > 0)
  • Trust floor Tmin = 0.1 prevents agent silencing
  • Trust ceiling Tmax = 0.9 prevents single-agent dominance
  • Session-scoped — resets between questions

4.5 Properties of Trust Dynamics

To ensure stability, the trust calibration mechanism exhibits the following theoretical properties. These are theoretical intuitions rather than formal convergence guarantees, consistent with the scope of a rigorous systems paper.

  • Bounded Trust Intervals: Trust is strictly bounded within [Tmin, Tmax] to prevent irreversible suppression or complete dominance.
  • Monotonicity: If an agent repeatedly produces evidence-supported claims, its trust converges monotonically toward Tmax.
  • Resistance to Collapse: Contradicted claims produce bounded trust decay. The trust floor prevents irreversible suppression of minority agents.
  • Minority Preservation Intuition: Even a single correct agent can outweigh a hallucinating majority if its evidence-grounded trust exceeds the combined decayed trust of the majority.
  • Probabilistic Interpretation: Trust Ti can be interpreted as a bounded empirical estimate of agent i's conditional correctness under evidence-grounded verification — i.e., P(agent i is correct | evidence retrieved so far).

4.5 Trust-Weighted Adjudication

Final answer is determined by trust-weighted aggregation over agent final positions, not majority vote. This is conceptually analogous to weighted ensemble methods but applied at the reasoning trace level — each agent's complete argument chain is weighted by its accumulated trust score Ti(K) at round K.

5.3 Experimental Protocol

  1. Primary Evaluation: Can trust-calibrated evidence-grounded aggregation reduce sycophantic consensus collapse? Measure baseline vs. trust-calibrated performance.
  2. Ablation Matrix: Evaluate the system by independently disabling key components:
    (a) Without trust calibration
    (b) Without progressive retrieval
    (c) Without source partitioning
    (d) Without adaptive triggering.
  3. Human Evaluation: Small-scale human evaluation (20–30 samples) reviewed by graduate students or domain experts. Measures reasoning quality, evidence quality, hallucination severity, and trustworthiness.

Statistical rigour: All experiments run across 3 random seeds. Results with 95% confidence intervals. Paired bootstrap significance testing (Koehn, 2004). Effect sizes (Cohen's d) reported alongside p-values.

📐 Formal Metric Definitions

MPR = |{debates where correct minority position survives to final output}| / |{total debates with correct minority}|

⚠ Risks & Mitigations
  • Compute cost: Multi-agent inference is N× more expensive. Mitigation: 4-bit quantized 8B models via vLLM; cap N=3, K=3. Cost per question ≈ 9 forward passes — tractable on a single A100 for 500-question evaluation.
  • RAG quality bottleneck: Noisy evidence degrades trust signal. Mitigation: Cross-encoder reranker; citation-count filtering (>10 citations); restrict to domain-specific abstracts.
  • Trust instability: Sigmoid-based update may converge trivially. Mitigation: Trust floor Tmin=0.1; stability analysis in ablations; compare against simpler heuristic (running accuracy) as sanity check.
  • Novelty concerns at review: Sycophancy papers are increasingly common. Mitigation: Frame as one of the first evidence-grounded trust-calibrated frameworks specifically targeting sycophantic consensus collapse; emphasise the novel CCR/MPR/ECR metrics as concrete evaluation contributions.

📌 10.5 Scope Boundaries

🛑 What This Work Does Not Claim

This work focuses specifically on scientific deliberation under adversarial consensus pressure and does not attempt to address general-purpose LLM alignment. Concretely, it does not claim to:

— Eliminate hallucinations (only reduce and detect them).
— Provide formal convergence proofs (only theoretical intuitions and empirical evidence).
— Generalise to all domains without retrieval infrastructure.
— Address long-term agent memory, planning, or general reasoning alignment.

🔍 10.6 Planned Failure Analysis

Systematic failure analysis is planned as part of Phase 4 (Apr–May 2027). Known risk scenarios and planned diagnostic approaches:

Correlated Hallucination

If all N agents share a common pre-training error, the trust function cannot detect it — all agents agree incorrectly. Diagnostic: compare homogeneous vs. heterogeneous agent configurations.

Retrieval Failure / Evidence Sparsity

For very recent or niche questions, no supporting evidence exists in corpora. Diagnostic: track abstention rate; report performance stratified by retrieval success rate.

Trust Collapse

Repeated contradictions early in a session may cause trust to approach Tmin for all agents. Diagnostic: monitor trust variance across rounds; trust floor prevents full collapse.

Adversarial Consensus

Two agents may converge on a confident but unverifiable claim, overwhelming a correct third agent. Diagnostic: track MPR under 2-vs-1 adversarial injection conditions.

1. Problem & Motivation

What exact problem are you solving? Most Important
We address the problem of agentic sycophancy in Multi-Agent Debate (MAD) systems, where LLM agents abandon correct initial positions to conform with a confidently-stated but incorrect majority opinion.

While debate systems are designed to improve reasoning quality, sycophancy causes the system to converge on confident hallucinations rather than evidence-grounded truth.

This becomes especially dangerous in domains such as: scientific reasoning, healthcare, clinical decision support, and scientific peer review.
Why is this problem important in real-world systems?
In high-stakes domains, an AI system that confidently converges on a wrong answer due to social conformity is often more dangerous than a system that simply abstains.

The specific failure mode is: a hallucinating majority suppressing a correct minority agent.

Existing multi-agent systems typically rely on: majority voting, which assumes consensus frequency corresponds to correctness. Our work challenges this assumption directly.
Why do current LLMs fail in scientific reasoning?
Current LLMs primarily rely on: parametric memory and probabilistic next-token prediction.

They lack:

1) Real-time evidence verification
2) Claim-level grounding
3) Resistance to social conformity under adversarial debate conditions

Huang et al. (2024) showed that LLMs cannot reliably self-correct reasoning without external feedback, which motivates our evidence-grounded trust mechanism.
Why is confidence calibration important?
Confidence calibration measures whether a model’s confidence aligns with its actual empirical correctness.

In our system, the trust score itself must be calibrated. Otherwise, highly trusted agents may still be systematically wrong.

We therefore evaluate calibration quality using: Expected Calibration Error (ECE).

2. Research Gap & Novelty

What is the research gap in existing literature? Most Important
Existing multi-agent debate systems improve reasoning quality, but most rely on: majority voting or equal-weighted aggregation.

They generally lack: evidence-grounded trust calibration, dynamic agent weighting, and explicit mechanisms for mitigating sycophantic consensus collapse.
What is your core novelty?
Our core contribution is an evidence-grounded trust calibration mechanism specifically designed to mitigate sycophantic consensus collapse in multi-agent deliberation systems.

The trust update mechanism is formally specified as:

Tᵢ(t+1) = Tᵢ(t) · σ( αVᵢ(t) − βHᵢ(t) )

where:
Vᵢ = evidence-supported claims
Hᵢ = contradicted claims
How is your work different from existing debate systems?
vs. MAD: Standard MAD uses majority voting. Our framework uses trust-weighted adjudication.

vs. ChatEval: ChatEval uses fixed roles. Our trust weights evolve dynamically based on evidence verification.

vs. CRITIC: CRITIC is single-agent. Our framework models cross-agent trust dynamics across debate rounds.

vs. PROClaim / iMAD: Existing systems lack explicit evidence-grounded trust weighting tied to agent influence.
Why is this research and not just prompt engineering?
Prompt engineering changes model instructions without introducing measurable algorithmic mechanisms.

Our work introduces: a mathematically specified trust update function, separable ablation components, and measurable evaluation metrics.

The contribution is algorithmic and experimentally falsifiable.

3. System Design & Architecture

How does your system work end-to-end? Most Important
The pipeline operates in four stages:

1. Adaptive Debate Triggering: A confidence estimator determines whether full debate is necessary.

2. Multi-Agent Deliberation: Multiple heterogeneous agents debate the scientific question.

3. Evidence Verification: Claims are decomposed into atomic propositions and verified through source-partitioned retrieval.

4. Trust-Weighted Adjudication: Final answers are aggregated using dynamically updated trust scores rather than majority voting.
Why not simply use a stronger single-agent model instead? Most Important
Stronger single-agent models improve overall capability, but they remain vulnerable to: hallucination, overconfidence, and unverified reasoning.

Multi-agent deliberation introduces: disagreement, critique, and independent evidence retrieval.

Importantly, consensus frequency is not equivalent to empirical correctness under adversarial sycophancy conditions.
Why is debate better than self-consistency prompting?
Self-consistency generates multiple reasoning paths from the same underlying model.

Debate introduces: adversarial critique, disagreement, counter-arguments, and iterative verification.

This produces stronger epistemic pressure against hallucinated reasoning.
How are trust scores calculated?
Trust scores are updated after each debate round using:

Tᵢ(t+1) = Tᵢ(t) · σ( αVᵢ(t) − βHᵢ(t) )

where:
Vᵢ = proportion of evidence-supported claims
Hᵢ = proportion of contradicted claims

Trust remains bounded within: [0.1, 0.9] to prevent collapse and single-agent dominance.
What happens if all agents are wrong?
This is a known limitation of all debate-based systems.

If all agents share the same misconception or retrieval failure, the trust mechanism may incorrectly reinforce false consensus.

We partially mitigate this through: heterogeneous model families, source-partitioned retrieval, and abstention handling when evidence is insufficient.
