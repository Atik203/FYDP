Trust-Calibrated Multi-Agent Debate
for Scientific Reasoning

LLM agents that argue, score each other's credibility, and resolve contradictions — with measurable trust calibration

Domain LLM + Agentic AI
Target Venue NeurIPS / ACL 2026
Risk Level ✅ Low
Compute Need Moderate
Overall Score 42 / 45
🔴
Real-World Problem
  • Scientific misinformation is accelerating across medical, climate, and policy domains
  • LLMs hallucinate plausible-sounding but factually incorrect scientific verdicts
  • No AI system can reason across multiple conflicting studies simultaneously
  • Clinicians and policymakers cannot trust AI-generated scientific summaries
  • Peer review is overwhelmed — 2.5M papers published annually
📚
Existing Solutions & Their Limitations
  • Multi-Agent Debate (Du et al., 2023): Agents debate but no trust scoring between them
  • SciFact / MultiVerS: Flat text matching, no argumentative structure
  • RAG Systems: Retrieve text but cannot resolve conflicting evidence
  • GPT-4 Scientific QA: Single-pass, no iterative reasoning or calibration
  • ChatPDF / Consensus.app: Surface-level — no contradiction resolution
🔬
Research Gap
  • No framework assigns dynamic trust scores to debating LLM agents
  • No calibration mechanism exists for multi-agent debate confidence
  • No system grounds agent debate in real scientific contradiction benchmarks
  • Agents in existing work are symmetric — no credibility weighting
  • Epistemic uncertainty output is absent from all current debate systems
✨
Our Proposed Solution
  • Trust Score Module: Dynamic credibility weights per agent per round
  • Calibrated Debate Protocol: Agents revise arguments based on trust feedback
  • Contradiction Resolution Layer: Explicit mechanism for conflicting claims
  • Epistemic Uncertainty Output: Verdict + confidence score, not just verdict
  • SciContra Dataset: New benchmark of contradictory scientific claim pairs
⚡
Novel Components (What Makes This New)
Trust Score Module (TSM)
First framework to assign dynamic credibility weights to LLM agents during debate — based on evidence quality and logical consistency
Calibrated Debate Protocol (CDP)
Agents iteratively revise arguments using trust-weighted feedback — a novel update mechanism beyond simple majority voting
Contradiction Resolution Layer (CRL)
Explicit neural module trained to identify and resolve conflicting scientific evidence across multiple retrieved papers
SciContra Benchmark
New publicly released dataset of scientific claim pairs with annotated contradictions — a standalone contribution for the community
⚙️
System Architecture & Pipeline
Input: Contradictory scientific claim pair ↓ Step 1: Retrieval Agent → fetches relevant papers (PubMed API + FAISS) ↓ Step 2: Agent A (Proponent) → argues FOR the claim with cited evidence Agent B (Opponent) → argues AGAINST with counter-evidence ↓ Step 3: Trust Scorer → evaluates each argument: [evidence grounding] + [logical consistency] + [source authority] → assigns Trust Weight: W_A, W_B ↓ Step 4: Debate Round 2 → agents revise arguments using trust feedback (repeat for N rounds until convergence or max rounds) ↓ Step 5: Contradiction Resolution Layer → synthesizes final verdict ↓ Step 6: Output: Verdict [Supported / Refuted / Contested] + Confidence Score [0.0 – 1.0] + Evidence Summary Models: Llama-3.1-70B / Mistral-Large (open-source) Framework: LangGraph + AutoGen Datasets: SciFact, MultiVerS, SciContra (custom built)

Argumentative Scientific
Claim Verification

NLP that understands argument structure — not just keyword matching — to verify complex scientific claims across contradictory literature

DomainNLP
Target VenueACL / EMNLP 2026
Risk Level✅ Low
Compute NeedLow-Medium
Overall Score36 / 45
🔴
Real-World Problem
  • 1M+ scientific papers published on PubMed annually — impossible to manually verify
  • The replication crisis: contradictory findings are common across disciplines
  • Misinformation citing "studies" spreads rapidly on social media
  • Current NLP fact-checkers treat claims as flat text — no argument structure
  • Meta-analyses take months/years — automated support is urgently needed
📚
Existing Solutions & Limitations
  • FEVER (Thorne et al., 2018): News fact-checking, not scientific argumentation
  • SciFact (Wadden et al., 2020): Single-sentence claims, no argument structure
  • MultiVerS (Wadden et al., 2022): Multi-document but no argumentative graph
  • Argument Mining (Stab & Gurevych): Persuasive essays only, not scientific text
  • GPT-4 Fact Checking: Hallucination-prone, no epistemic calibration
🔬
Research Gap
  • No system jointly models claim → evidence → rebuttal → qualifier chains
  • Cross-study contradiction resolution is completely unaddressed
  • No epistemic uncertainty quantification based on study design quality
  • No graph-based representation of scientific argumentative discourse
  • No benchmark for cross-paper contradictory scientific claim pairs
✨
Our Proposed Solution
  • Scientific Argument Graph (SAG): Nodes = claims, evidence, rebuttals; Edges = supports/contradicts/qualifies
  • Contradiction-Aware Verifier (CAV): Trained on conflicting evidence pairs
  • Epistemic Uncertainty Scorer: Weights by study design (RCT > observational, sample size, p-values)
  • SciContra Dataset: New benchmark of annotated contradictory claim pairs
  • GNN Reasoning: Graph Neural Network over SAG for structured reasoning
⚙️
System Pipeline
Input: Scientific claim to verify (e.g., "Vitamin D prevents COVID-19") ↓ Step 1: Retrieve relevant papers via SciBERT encoder + FAISS index ↓ Step 2: Build Scientific Argument Graph (SAG) Nodes: [Claim] [Evidence] [Rebuttal] [Qualifier] Edges: supports → contradicts → qualifies → ↓ Step 3: Graph Neural Network (GNN) reasons over SAG → learns which evidence paths are most credible ↓ Step 4: Epistemic Scorer weights by study design quality: RCT (5) > Cohort (4) > Case Study (3) > Opinion (1) + sample size + statistical significance (p-value) ↓ Step 5: Contradiction-Aware Verifier synthesizes cross-study verdict ↓ Output: [Supported / Refuted / Contested] + Confidence Score + Evidence Chain Explanation Models: SciBERT, PubMedBERT, PyTorch Geometric (GNN) Datasets: SciFact, CORD-19, MultiVerS, SciContra (custom)

Temporal Knowledge Decay
+ Dynamic Knowledge Graphs

When AI knowledge goes stale — detect it, score it, and resolve conflicts between old memory and new evidence

DomainNLP
Target VenueEMNLP / AI Journal
Risk Level✅ Very Safe
Compute NeedLow-Medium
Overall Score32 / 45
🔴
Real-World Problem
  • Medical NLP systems trained in 2022 miss 2024 drug guidelines — patient safety risk
  • Legal NLP misses new regulations, case law, and statutory changes
  • Financial NLP has outdated company profiles, market conditions, leadership changes
  • Full LLM retraining costs millions — not viable for continuous updates
  • RAG retrieves fresh text but cannot resolve conflicts with old parametric knowledge
📚
Existing Solutions & Limitations
  • RAG (Lewis et al., 2020): Retrieves fresh text but ignores temporal validity
  • TempLAMA benchmark: Evaluates retrieval accuracy, not conflict resolution
  • ROME / MEMIT (knowledge editing): Modifies weights but doesn't scale to continuous drift
  • TemporalWiki: Dataset exists but no conflict resolver built on it
  • Periodic retraining: Expensive ($millions), not continuous, loses old knowledge
🔬
Research Gap
  • No metric exists to score "how stale" a piece of retrieved knowledge is
  • No framework detects conflicts between parametric memory and retrieved documents
  • Temporal validity of retrieved documents is universally ignored in RAG pipelines
  • No benchmark specifically for temporal knowledge conflict resolution in NLP
  • Domain-specific temporal decay rates (medical vs. legal vs. financial) are unstudied
✨
Our Proposed Solution
  • Temporal Validity Score (TVS): Novel metric — time-decay weight for each retrieved document
  • Dynamic Knowledge Conflict Resolver (DKCR): Detects + resolves parametric vs. retrieved conflicts
  • Temporal Reasoning Chain: "What was true when" explicit reasoning over time-stamped KG
  • TempoConflict Benchmark: New dataset of conflicting temporal knowledge pairs
  • Domain-Specific Decay Models: Medical / Legal / Financial temporal decay rates
⚙️
System Pipeline
Input: Time-sensitive query (e.g., "current recommended dosage for Drug X") ↓ Step 1: Retrieve documents + timestamps from knowledge base ↓ Step 2: Temporal Validity Scorer computes TVS for each doc: TVS = f(recency_weight, domain_decay_rate, source_authority) Medical: high decay rate | Legal: medium | General: low ↓ Step 3: Conflict Detector: Compare parametric LLM memory vs. retrieved documents Flag: [No Conflict] / [Soft Conflict] / [Hard Conflict] ↓ Step 4: DKCR — Dynamic Knowledge Conflict Resolver: IF Hard Conflict → prioritize highest TVS document IF Soft Conflict → weighted fusion of parametric + retrieved IF No Conflict → standard generation ↓ Step 5: Temporal Reasoning Chain: "As of [DATE], [ENTITY] had [PROPERTY]. This changed in [DATE] to [NEW_PROPERTY]." ↓ Output: Answer + temporal confidence + conflict resolution trace Models: DeBERTa-v3, T5, Wikidata Temporal API Datasets: TempLAMA, TemporalWiki, TempoConflict (custom)

Neuro-Symbolic Rare
Disease NLP

Combining neural language understanding with symbolic medical ontology reasoning to diagnose rare diseases from clinical text — even with almost no training data

DomainNLP + Healthcare
Target VenueJAMIA / ACL BioNLP
Risk Level✅ Manageable
Compute NeedModerate
Overall Score32 / 45
🔴
Real-World Problem
  • 300 million people worldwide are affected by rare diseases
  • Average diagnosis takes 4–7 years — misdiagnosis is the norm, not exception
  • Clinical notes describe symptoms in unstructured, ambiguous natural language
  • Pure neural models fail catastrophically with sparse data (rare = few cases)
  • Symbolic ontologies exist but are disconnected from modern NLP pipelines
📚
Existing Solutions & Limitations
  • BioBERT / PubMedBERT: Require large training datasets — rare diseases have almost none
  • UMLS / SNOMED-CT / ORDO: Rich structured knowledge but not NLP-integrated
  • GPT-4 Medical QA: Strong generally but hallucination-prone for rare cases
  • Clinical NLP tools (MetaMap, cTAKES): Rule-based, cannot reason across symptoms
  • Few-shot medical models: No symbolic grounding, error-prone for rare diseases
🔬
Research Gap
  • No framework integrates neural NLP + medical ontology for rare disease reasoning
  • Few-shot learning for rare diseases is unsupported by symbolic grounding
  • Explanation generation for rare disease diagnosis is completely absent
  • UMLS and ORDO knowledge is not injected into transformer attention mechanisms
  • No benchmark specifically for rare disease NLP from clinical notes
✨
Our Proposed Solution
  • Ontology-Grounded Encoder: Injects UMLS/ORDO knowledge into transformer attention layers
  • Symbolic Reasoning Layer: Rule-based diagnostic reasoning over ontology paths
  • Few-Shot Rare Disease Classifier: Meta-learning for diseases with <50 training examples
  • Explanation Generator: Human-readable diagnostic reasoning chains for clinicians
  • Neural-Symbolic Fusion: Combines neural confidence + symbolic diagnostic rules
⚙️
System Pipeline
Input: Clinical note (unstructured text from EHR) ↓ Step 1: Neural Encoder (PubMedBERT) extracts symptom entities Named Entity Recognition → [fever, joint pain, skin rash, fatigue] ↓ Step 2: Ontology Grounding Map entities → UMLS Concept IDs → ORDO Rare Disease Paths e.g., [skin rash + fever + joint pain] → Lupus candidate path ↓ Step 3: Symbolic Reasoning Layer Traverse ontology graph: symptom → disease candidate Apply diagnostic rules (IF symptom_set THEN disease_candidate) Confidence: path_length + ontology_coverage_score ↓ Step 4: Few-Shot Neural Classifier (Meta-Learning) Prototypical Networks over disease embeddings Handles diseases with < 50 training examples ↓ Step 5: Neural-Symbolic Fusion Final_Score = α × Neural_Score + β × Symbolic_Score ↓ Step 6: Explanation Generator "Patient shows [symptoms] which align with [disease] via UMLS path: [symptom→mechanism→disease]" ↓ Output: Ranked rare disease candidates + explanation chains Models: PubMedBERT, Prototypical Networks, Py2neo Datasets: MIMIC-III, RareDis corpus, OMIM, ORDO

Scientific Verification
Agentic LLMs

A multi-agent LLM pipeline that autonomously decomposes, retrieves, audits, and verifies scientific claims — with a hallucination guard

DomainLLM + Agentic AI
Target VenueACL / NeurIPS 2026
Risk Level⚠️ Scope Carefully
Compute NeedMedium-High
Overall Score31 / 45
🔴
Real-World Problem
  • 2.5 million scientific papers published annually — manual verification is impossible
  • LLMs confidently hallucinate scientific verifications — dangerous in medicine/policy
  • No AI system can autonomously cross-check experimental claims in a paper
  • Peer review takes 6–18 months — automated assistance could transform science
  • Fraudulent papers with plausible-sounding methodology go undetected
📚
Existing Solutions & Limitations
  • AI Scientist (Lu et al., 2024): Generates papers — does NOT verify claims
  • GPT-4 Review Assistant: Single-pass, no tool use, high hallucination rate
  • Semantic Scholar AI: Summarization only — no claim-level verification
  • Scite.ai: Citation context analysis — not autonomous verification
  • PaperQA2: QA over papers — no methodology auditing capability
🔬
Research Gap
  • No system decomposes papers into atomic verifiable claims autonomously
  • No agentic framework combines retrieval + methodology audit + hallucination guard
  • Hallucination control in multi-step scientific reasoning is unsolved
  • No structured verification report format exists for automated scientific review
  • Tool-using LLMs for scientific verification are entirely unstudied
✨
Our Proposed Solution
  • Claim Decomposition Agent: Breaks paper into atomic verifiable claim units
  • Evidence Retrieval Agent: Searches PubMed / Semantic Scholar for supporting/contradicting evidence
  • Methodology Auditor Agent: Checks statistical validity, sample size, p-value reporting
  • Hallucination Guard: Confidence-gated — refuses to verify without sufficient evidence
  • Verification Report Generator: Structured human-readable verification summary
⚙️
Multi-Agent System Pipeline
Input: Scientific paper / abstract (biomedical domain — scoped for FYDP) ↓ Agent 1: Claim Decomposition Agent Paper → [Claim_1, Claim_2, ..., Claim_N] e.g., "Drug X reduces mortality by 30% in RCT" ↓ For each Claim_i: Agent 2: Evidence Retrieval Agent PubMed API + Semantic Scholar API search Returns: [Supporting_papers] + [Contradicting_papers] ↓ Agent 3: Methodology Auditor Agent Checks: study_type, sample_size, p_value, effect_size Flags: [Valid] / [Questionable] / [Invalid] ↓ Agent 4: Hallucination Guard IF confidence < threshold → "Insufficient evidence to verify" IF confidence ≥ threshold → proceed to verdict ↓ Claim Verdict: [Supported / Contradicted / Insufficient / Contested] ↓ Agent 5: Synthesis + Report Generator Structured Verification Report: | Claim | Status | Evidence Count | Confidence | Issues | ↓ Output: Complete verification report + per-claim evidence chains ⚠️ FYDP SCOPE: Biomedical abstracts only | 3 agents max | Abstract-level Models: Llama-3.1-70B / Mistral-Large Framework: LangGraph, GROBID (PDF parsing), PubMed API Datasets: SciEval, ExpertQA, PeerRead, SciFact
